          • Installation
            Download the agilesites-bigbang.zip from www.agilesites.org/bigbang.zip and unzip it.

            Copy agilesites.properties.dist into agilesites.properties.

            Edit it to match your system. Most important you have to set:

            • java: java executable you want to use
            • site.focus : the name of the main site of your project. It is recommended it is a lower case name
            • sites.directory is an empty writable directory where to install sites

            On Windows, go to the command prompt and type agilesites.cmd. On Linux/O...

          • Create a new site
            The first step to implement a new site with AgileSites is to create a new site.

            You do that creating a class and adding an annotation

            import agilesites.*;
            import agilesites.annotations.*;
            import static agilesites.Api.*;

            @Site
            class BigBang extends Site {

            Site() {
            addType(“Jar”)
            }

            }

        • Layout
          AgileSites11g is the meta project, it has subproject:

          - Lib (with core and api)
          - Plugin (the plugin)
          - Setup (the installation kit)
          - BigBang (the demo project)

          The MetaProject can:

          - Import/Export the AdminSite
          - Deploy the setup.jar in the setup project
          - Initialize a repo for a Standalone app
          (running with sbt with a preloaded repository)
          - Execute subproject the plugin imported in source format (no need to publish/reload)

          • AAAgileSetup
            AAAgileSetup is the setup command

            The complete syntax is

            GET /cs/ContentServer?pagename=AAAgileSetup&op=XXX&etc

            I will omit the initial part and focus on the op and the parameters:

            It will interpret the following commands:

            GET …op=hello

            Returns a cookie to initialize the session

            GET …op=init&site=XXX&siteid=YYY

            It will create Jar and Static basic types (it not there)
            It will initialized the Site with Siteid
            Return an authentication key and a list of existing Jars

            GET …op=upload
